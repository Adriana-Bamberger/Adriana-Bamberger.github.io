JavaScript and its relationship to HTML and CSS.

To help explain the three's connection we'll use an analogy.
Say we're building a playspace for our imaginary pet hamsters,
The HTML is the main structure, it's the tunnels, wheel, hammocks crawl spaces etc', And the cage that holds it all.
The CSS is the paint we've used, and the coloured dye we've used on the different pieces to make it look visually pleasing.
The JavaScript is the ball bearings we've used to make the wheel spin around and around, the parts that make the automatic feeder and drink station work.
Thus, in our website,
HTML provides the base, CSS makes it pretty, and JavaScript adds the interactive elements that keep the people (and hamsters!) entertained.

Control flow and loops

Are key parts of any webpage,
We'd use control flow as such; Say our user has been invited to a party, they’re standing in front of our “closet”, This would be our application or website UI,
It's full of clothes, these are our options, we call options “conditions”,
Is it a fancy dinner party or a casual catch up?

If yes it's a Fancy dinner, we would direct them to grab a random dress or jumpsuit.
If not, it must be a casual catch up. We'd direct them to pick a nice pair of jeans and a fun top.

Now, the user has some clothes, but we still need shoes,
This would be our loops:
First we have an action, Try on a pair of shoes,
Do these shoes match? Our condition,
If Yes, Great, you're done!
If no, loop, Back to the shoe rack our user goes to try another pair until we have found a pair that does eventually match.

And when we use them both at the same time; we can go looking for a matching necklace and earring set.
User looks in a Jewelry box, looking for a matching necklace and earrings set.
This would be our condition.
If yes, Put them both on and you're done!
If no, look again.
Our user has checked if the jewelry box has a matching set (control flow), and if not, the user has to repeat the search (loop) until finding one.

What is the Dom?

The Dom could be like a map of a house.
It breaks down the entire house (webpage) into its individual parts,
For example; walls, door frames, doors, cupboards, stairs, ceilings and floors, (headings, paragraphs, images, and buttons)
Each part becomes an object you can access and manipulate with Javascript.
We call these parts “nodes”

Some of the things we can do with nodes are,

  • Change or update text on the page (e.g., button labels, headings). “InnerHTML” is one of the ways of doing this.
  • Modify parts styles, changing colors, fonts, or other visual aspects of elements.
  • Add / create or Remove Elements from the page.
  • Show or hide elements with user actions/events.
  • Build buttons, menus, or other interactive components.
  • Real-time information like social media feeds.
  • Animations and transitions on the page.

The difference between accessing data from arrays and objects.

Let's imagine I'm planning a party. I've used an array to list out all the people I have invited.

Each person has a specific position/number identifier in our list (array).
However, unlike what we've traditionally been taught, we start counting from zero.
The first person is at position 0, the second at position 1, and so on.
These positions are the indexes in an array.

To find a specific friend, I need to know their exact position (index) in the line.

Objects are like boxes filled with all my party decorations.
Each item has a specific label describing it. Not a number.
I'd have a label for "balloons", "tablecloths", " streamers", etc.
These labels are the properties in an object.

When I want to find out how many friends are coming to my event I go to our array, find the highest index number and "+ 1" as we've started counting from zero.

To find a specific friend, I can call out their position using their index number. If she's the 5th person I invited, she'll be in index number four.

To find out what kind of cake I bought (an object), I'll look for the label "Cake" in the box and see whatever kind of cake I've put in that label.

Basically, arrays are like ordered lists where you find things by their position number.
Objects are like labeled boxes where you find things by their specific name.

What are Functions?

Functions are boxes that hold blocks of code/instructions.
They can be "called upon" or referenced multiple times.
They take inputs, we call them “parameters”, then process them, and can sometimes return an output.

They're helpful as they force you to break down complex problems into smaller chunks.
They make things easier to understand and read, easier to change and edit in the future, and easier to test + debug.
